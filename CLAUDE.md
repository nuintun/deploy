## Project Overview

`@nuintun/deploy` is a CLI tool and library for publishing project files via FTP and/or SVN. It supports single or multi-task deployment configs, glob-based file filtering, dry-run preview, and clean-before-deploy for both protocols.

## Commands

```bash
# Type-check (lint)
pnpm lint

# Full build (ESM + CJS + fix .d.ts paths + format)
pnpm build

# Build ESM only
pnpm build:esm

# Run tests (builds first via pretest, then runs manual test suite)
pnpm test

# Format all files
pnpm format

# Clean build output
pnpm clean
```

There is no watch/dev mode. The test file is [tests/index.ts](tests/index.ts) — a manual test runner, not a test framework. Add new test cases inline following the existing pattern.

## Architecture

**Pipeline: Config → Plan → Execute**

1. **CLI** ([src/cli.ts](src/cli.ts)) parses args (`--config`, `--dry-run`, `--message`, `--help`, `--version`), loads a config file via dynamic `import()`, and calls `deploy()`.
2. **`deploy()`** ([src/index.ts](src/index.ts)) iterates over tasks (single or array), runs the planner then the executor for each.
3. **`DeployPlanner`** ([src/planner/deploy-planner.ts](src/planner/deploy-planner.ts)) stats each entry's source, builds `Operation` objects with the correct `OperationType`, and returns a `DeployPlan`. Filters (`include`/`exclude` globs) are compiled to regex here.
4. **`DeployExecutor`** ([src/executor/deploy-executor.ts](src/executor/deploy-executor.ts)) lazily creates and caches transport adapters per unique key, then runs operations sequentially. On first failure it stops and cleans up all adapters in a `finally` block.
5. **Transport adapters** implement the `TransportAdapter` interface ([src/adapters/transport-adapter.ts](src/adapters/transport-adapter.ts)):
   - **`FtpAdapter`** ([src/adapters/ftp-adapter.ts](src/adapters/ftp-adapter.ts)) wraps `basic-ftp`'s `Client`. Connects on first use, caches ensured directories to avoid redundant `ensureDir` calls.
   - **`SvnAdapter`** ([src/adapters/svn-adapter.ts](src/adapters/svn-adapter.ts)) shells out to the `svn` CLI. Accumulates pending paths, runs `svn add` only for unknown (`?`) files, and commits only if there are actual changes.

**Key types** are in [src/types/config.ts](src/types/config.ts) (user-facing config shapes) and [src/types/operation.ts](src/types/operation.ts) (internal operation discriminated union with `OperationType` const enum).

**Utilities:**

- [src/utils/path.ts](src/utils/path.ts) — all path operations normalize to POSIX (forward slashes). Use these instead of `node:path` directly.
- [src/utils/glob.ts](src/utils/glob.ts) — custom glob-to-regex compiler supporting `*`, `?`, and `**` (globstar). Case-insensitive on Windows.
- [src/utils/file-walker.ts](src/utils/file-walker.ts) — async generator that recursively walks directories with filter support and optional symlink following.
- [src/utils/logger.ts](src/utils/logger.ts) — structured logger with levels controlled by `DEPLOY_LOG_LEVEL` env var, color controlled by `DEPLOY_NO_COLOR`.

## Build System

Rollup with `@rollup/plugin-typescript` builds from `src/` to `esm/` and `cjs/`. After the ESM build, `tools/fix-types.ts` runs `dts-paths` to rewrite `/*` path aliases in `.d.ts` files to relative paths. The CJS build is generated by the same rollup config with `esnext=false` flag.

TS path alias `/*` → `./src/*` is used for internal imports. The published package remaps these, so the alias only works at dev time.

## Code Style

See [.agents/skills/code-style/SKILL.md](.agents/skills/code-style/SKILL.md) for detailed conventions. Key points:

- Private class fields use `#prefix` syntax
- Imports sorted by line length (ascending), then alphabetically case-sensitive
- `const enum` for compile-time constants (e.g., `OperationType`)
- Type guards preferred over non-null assertions or type casts
- JSDoc on public APIs only; inline comments explain why, not what
- Module headers: `/** @module module-name */`

Formatting is enforced by Prettier (128 print width, single quotes, no trailing commas, 2-space indent, LF).
